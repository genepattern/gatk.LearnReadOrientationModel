# gatk.LearnReadOrientationModel (v1)

**Description**: Learns read orientation artifact priors from F1R2 counts produced by Mutect2 or CollectF1R2Counts. Uses an Expectation-Maximization algorithm to model orientation-dependent sequencing artifacts at each trinucleotide context. Output is used by FilterMutectCalls to filter orientation bias artifacts such as OxoG damage.

**Authors**: Broad Institute of MIT and Harvard; GenePattern Team, UC San Diego

**Contact**: [GenePattern Community Forum](https://groups.google.com/g/genepattern-help)

**Algorithm Version**: GATK 4.1.4.1 LearnReadOrientationModel

## Summary

**gatk.LearnReadOrientationModel** is step 3 in the GATK4 Best Practices somatic variant calling pipeline. It takes one or more F1R2 counts files (tar.gz archives generated by Mutect2 with `--f1r2-tar-gz` or by CollectF1R2Counts) and learns the rates of read orientation artifacts at each trinucleotide context using an Expectation-Maximization (EM) algorithm.

Read orientation artifacts arise when the two strands of a DNA duplex are sequenced asymmetrically, typically due to oxidative damage (OxoG) during library preparation. These artifacts appear predominantly on reads in one orientation (F1R2 or F2R1) relative to the reference strand. The EM algorithm estimates the prior probability that each trinucleotide context produces an artifact, which FilterMutectCalls uses to compute a posterior probability that an observed variant is artifactual.

**Typical pipeline position:**

```
Mutect2 (--f1r2-tar-gz output)
         |
         v
gatk.LearnReadOrientationModel  ---> artifact-prior.tar.gz
         |
         v
    FilterMutectCalls (--orientation-bias-artifact-priors)
```

**Multiple scatter inputs:** When Mutect2 is run in scatter-gather mode (one job per genomic interval), each scatter produces its own F1R2 tar.gz. Provide all scatter outputs as multiple `input.tar.gz` values; the tool combines them into a single learned model before running the EM algorithm.

## Input Files

1. **input tar.gz** *(required, one or more)*
   One or more F1R2 counts tar.gz files produced by Mutect2 (via `--f1r2-tar-gz`) or by the CollectF1R2Counts tool. Each file contains per-trinucleotide-context counts of alt sites and ref site depth histograms, separated by read orientation (F1R2 vs F2R1).

   When Mutect2 is run in scatter mode (one job per genomic interval), each scatter produces a separate F1R2 tar.gz file. Provide all scatter files as multiple `input.tar.gz` values; the tool aggregates counts across scatters before fitting the model. Passed to GATK as `-I / --input`.

   - **Format**: tar.gz archive (.tar.gz or .gz)
   - **Source**: Mutect2 `--f1r2-tar-gz` output, or CollectF1R2Counts output

2. **arguments file** *(optional)*
   A plain-text file containing additional GATK command-line arguments, one per line or as `argument=value` pairs. Use this to specify advanced LearnReadOrientationModel options not exposed as dedicated module parameters (e.g., `--java-options -Xmx6g`, `--tmp-dir /scratch`). Passed to GATK via `--arguments_file`.
   - **Format**: Plain text (.txt)

3. **gatk config file** *(optional)*
   A GATK configuration file in Java properties format. Allows customization of GATK runtime behavior such as logging verbosity, performance tuning, and default argument values. Passed to GATK via `--gatk-config-file`.
   - **Format**: Properties file (.properties, .conf, .config, or .txt)

## Output Files

1. **Artifact prior tar.gz** (filename specified by **output file name**, default: `artifact-prior.tar.gz`)
   A tar.gz archive containing per-trinucleotide-context artifact prior tables learned by the EM algorithm. This file is passed directly to FilterMutectCalls via its `--orientation-bias-artifact-priors` parameter to enable orientation bias artifact filtering.

   The archive contains one table per sample, with the artifact prior probability for each combination of trinucleotide context and mutation type (e.g., the rate of C>A OxoG artifacts in ACT context).

## Requirements

- **Docker image**: `broadinstitute/gatk:4.1.4.1`
- **Runtime environment**: GenePattern server with Docker support
- **Memory**: Minimum 4 GB RAM; 8 GB or more recommended. Java heap size can be adjusted via `--java-options -Xmx<N>g` in the **arguments file**.
- **Input format requirements**:
  - Input F1R2 tar.gz files must be produced by Mutect2 or CollectF1R2Counts from the same GATK version
  - All input scatters must be from the same sample (or samples, if multi-sample calling was used)
- **Note**: Unlike other GATK tools in this pipeline, LearnReadOrientationModel does not require a reference genome or companion index files.
